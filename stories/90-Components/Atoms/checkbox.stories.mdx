# Checkbox

Checkboxes are elements used to let a user pick one or more options from a set number of options. Selected options display with a checkmark and solid fill background. Clicking a checkbox selects or deselects it depending on the initial state.
  

## Use Cases

  
**Terms and conditions**
Selecting a checkbox can indicate whether you agree to the terms. This implementation can be seen on Gothamist inside Sign Up Forms.

  

**Forms**
Can be used in forms to select options.

  

**Filtering**
Can be used to select different filter options for filtering a list of data.

  
  

## Checkbox vs. Radio Button

If a user can only select **one option from a list**, use radio buttons, which are mutually exclusive. If a user can select **more then one option from a list** use check boxes. If there’s only **one option**, use a checkbox.
  
  
  
  

## What’s Inside

**Checkbox Input** - A checkbox input indicating selected or unselected. By default, it is unselected.

**Checkbox Label** - A short string describing the option.

## Alignment

Checkbox labels should be positioned to the right of their inputs. For more than one option, checkboxes can be arranged horizontally or vertically. When possible, arrange checkbox options groups vertically for better scannability.

  
  

## Accessibility

-   Checkboxes labels and titles should be clear and concise.
    
-   Warn users if selecting a checkbox will cause an on-screen change.
    
-   Add `aria-required` for required inputs.
